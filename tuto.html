Créer un site web — guide pas à pas

Ce document t'explique exactement quoi dire et quoi montrer pendant ton exposé. Chaque étape contient : explication simple, exemples, et points à présenter.

Étape 1 — Idée & plan (ce que tu dois dire)

Pourquoi ? Expliquer l'objectif du site — sans objectif clair, le site n'a pas de direction. Montre un croquis (dessin rapide) : c'est très parlant pour l'auditoire.

Points à aborder à l'oral

  • Quel est le but : (ex : portfolio pour montrer mes projets / blog pour partager / page d'association)
  • Quels visiteurs ? (amis / recruteurs / fans d'anime)
  • Quelles pages : Accueil, À propos, Tutoriel, Projets, Contact
  • Wireframe : header (menu), zone principale, footer
  • Choix stylistiques : palette, police, ambiance

Phrase prête à dire

« J'ai choisi de créer un site pour présenter mes projets : il y aura une page d’accueil, une page tutoriel, une page projets, et un footer avec mes crédits. »

Exercice rapide (sur scène)

Demande au public : « Si je devais ajouter une page, laquelle ? » — note les réponses et explique pourquoi tu choisis ou non.

Étape 2 — HTML : structure & sémantique

But : montrer que HTML n'est pas du style, mais la structure. C'est le squelette.

Code simple (montrer à l'écran)

<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="utf-8">
  <title>Mon site</title>
</head>
<body>
  <header>Menu / logo</header>
  <main>
    <section>Accueil</section>
    <section>Projets</section>
  </main>
  <footer>Crédits</footer>
</body>
</html>

Points à expliquer

  • <head> : meta, titre, liens CSS — sert aux navigateurs et moteurs de recherche.
  • <header> et <footer> : zones standards, utiles pour l’accessibilité.
  • Utiliser des balises sémantiques aide le référencement (SEO) et l’accessibilité.

À montrer en live

Ouvre un simple fichier HTML dans ton navigateur : montre que sans CSS c’est brut mais lisible — c’est pourquoi on ajoute ensuite du CSS.

Étape 3 — CSS : mise en forme, grilles et responsive

But : rendre le site agréable visuellement et lisible sur téléphone et PC.

Exemples concrets

/* police, fond et couleur */
body {
  font-family: 'Poppins', sans-serif;
  background: #06060a;
  color: #fff;
}
/* layout */
.container { max-width: 1000px; margin: 0 auto; padding: 20px; }

Points à expliquer

  • Palette : 2-3 couleurs → cohérence visuelle
  • Spacing (padding/margin) : respirer le contenu
  • Responsive : media queries — montre le même site sur petit écran
  • Transitions/hover : petites animations pour rendre interactif

Démo pour l’exposé

Montre « avant/après » : même HTML, sans CSS puis avec CSS. Le contraste fait son effet.

Étape 4 — JavaScript : interactions utiles

JS ajoute le dynamisme : validation formulaire, menu mobile, animations.

Exemple simple à montrer

document.querySelector('.btn').addEventListener('click', function() {
  alert('Bienvenue dans mon tutoriel !');
});

Points à expliquer

  • Event listeners (clics, scroll)
  • Manipulation du DOM (changer le HTML/CSS depuis JS)
  • Ne pas abuser : garder les interactions utiles

Mini-démo

Affiche un bouton et fais une alerte, puis montre comment remplacer l'alerte par une animation douce.

Étape 5 — Publier ton site (mise en ligne)

Tu peux publier gratuitement un site statique (HTML/CSS/JS). Pour ton exposé, montre la page en local, puis la version en ligne.

Processus simple (à dire)

  1. Créer un dépôt GitHub.
  2. Ajouter les fichiers (index.html, style.css, script.js, tuto.html).
  3. Activer GitHub Pages (Settings → Pages → main branch → root).
  4. Accéder à ton site : https://tonpseudo.github.io/

Astuce exposé

Montre une capture « avant/après » : modifications locales → push → site mis à jour.

Étape 6 — Améliorer : SEO, performances, accessibilité

  • Optimise les images (WebP, compression).
  • Ajoute des titres et meta description pour le référencement.
  • Vérifie le contraste et les attributs alt pour l’accessibilité.
  • Testes la vitesse (GTmetrix ou Lighthouse) et corrige les points faibles.

Conclusion prête à dire

« Créer un site, c’est une combinaison d’idée, d’organisation, d’un squelette HTML, d’un beau style CSS, et d’un JS qui rend le tout vivant. »

Questions fréquentes à préparer

  • Comment choisir un nom de domaine ?
  • Quelle différence entre hébergement gratuit et payant ?
  • Comment rendre le site mobile friendly ?

Checklist pour ton exposé (copie/colle dans tes notes)

  1. Présenter l’objectif du site et le public visé.
  2. Montrer le wireframe (croquis).
  3. Expliquer la structure HTML.
  4. Afficher le « avant/après » CSS.
  5. Faire une petite démo JS (bouton ou fond animé).
  6. Montrer la version en ligne (GitHub Pages).
  7. Répondre aux Q/A préparées.
